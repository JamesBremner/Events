void cTimedEvent::handle_timer_event ( )
inlinevirtual

Function to post when the timer fires

This demonstrates that the timers run asynchronously ( in parrallel and without interfering with the executing application code ) but the event handlers run synchronously.

The output should show that each event handler runs to completion once started before the next event handler starts, even though the event handler reuire 1 second to complete, but are schedule to run ever 300 msecs.

The advantage of this is that the application code need not worry about contention between the different handlers, no matter when thet are posted to run.
